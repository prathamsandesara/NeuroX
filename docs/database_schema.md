# 🗄️ Database Schema

## Overview
NeuroX uses a normalized relational database (PostgreSQL via Supabase) to ensure data integrity and efficient querying.

## Entity Relationship Diagram (ERD) - Conceptual
- **One User** can have **Many Jobs** (as Recruiter) or **Many Submissions** (as Candidate).
- **One Job** has **One Assessment**.
- **One Assessment** has **Many Questions**.
- **One Submission** belongs to **One User** and **One Assessment**.
- **One Submission** has **One Result**.

## Table Definitions

### 1. Users (`users`)
Stores account credentials and profile info.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier. |
| `email` | TEXT | Unique email address. |
| `password_hash` | TEXT | Bcrypt hash. |
| `role` | ENUM | `ADMIN`, `RECRUITER`, `HR`, `CANDIDATE`. |
| `is_verified` | BOOLEAN | Email verification status. |
| `otp_hash` | TEXT | Temporary hash for OTP verification. |
| `created_at` | TIMESTAMPTZ | Creation timestamp. |

### 2. Jobs (`jobs`)
Stores job openings created by recruiters.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier. |
| `title` | TEXT | Job Title (e.g., "React Developer"). |
| `skills` | JSONB | List of required skills/weights. |
| `difficulty_level` | ENUM | `JUNIOR`, `INTERMEDIATE`, `SENIOR`, `EXPERT`. |
| `experience_min` | INTEGER | Minimum years required. |
| `experience_max` | INTEGER | Maximum years required. |
| `created_by` | UUID (FK) | Reference to `users(id)`. |

### 3. Assessments (`assessments`)
Links questions to a job.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier. |
| `job_id` | UUID (FK) | Reference to `jobs(id)`. |
| `assessment_distribution` | JSONB | Config for question types `{ "mcq": 30, ... }`. |

### 4. Questions (`questions`)
Stores individual questions generated by AI.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier. |
| `assessment_id` | UUID (FK) | Reference to `assessments(id)`. |
| `type` | ENUM | `MCQ`, `SUBJECTIVE`, `CODING`. |
| `content` | JSONB | Stores question text, options, correct answers, test cases. |
| `difficulty` | TEXT | Difficulty level relative to job. |

### 5. Submissions (`submissions`)
Tracks a candidate's attempt at an assessment.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier. |
| `user_id` | UUID (FK) | Reference to `users(id)`. |
| `assessment_id` | UUID (FK) | Reference to `assessments(id)`. |
| `score` | DECIMAL | Final aggregate score. |
| `status` | TEXT | `IN_PROGRESS`, `COMPLETED`. |
| `attempts_left` | INTEGER | Remaining retries (Default 1). |
| `started_at` | TIMESTAMPTZ | Start time. |
| `completed_at` | TIMESTAMPTZ | End time. |

### 6. Results (`results`)
Detailed breakdown of a submission.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier. |
| `submission_id` | UUID (FK) | Reference to `submissions(id)`. |
| `details` | JSONB | Full answer breakdown, per-question scores. |

### 7. Integrity Logs (`integrity_logs`)
Stores AI-driven cheating analysis.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier. |
| `user_id` | UUID (FK) | Reference to `users(id)`. |
| `assessment_id` | UUID (FK) | Reference to `assessments(id)`. |
| `risk_score` | DECIMAL | 0-100 Score. |
| `risk_level` | TEXT | `LOW`, `MEDIUM`, `HIGH`. |
| `flagged` | BOOLEAN | If manual review is recommended. |
| `details` | JSONB | Metrics like tab switches, browser focus time. |
